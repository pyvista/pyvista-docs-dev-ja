pyvista.core.input_validation.check#

入力のタイプと/または値をチェックする関数．

バージョン 0.43.0 で追加.

check 関数は通常，次のように動作します:

  • 単一の入力変数に対して単純な検証を行う．

  • 無効な入力によりチェックに失敗した場合，エラーを発生させます．

  • 入力を変更したり，何も返したりしません．

関数

check_has_length(arr, /, *[, exact_length, ...])

配列の長さが特定の条件を満たすかどうかをチェックします．

check_has_shape(arr, /, shape, *[, name])

配列が指定された形状を持つかどうかをチェックします．

check_is_arraylike(arr)

入力がNumPyのndarrayとしてキャストできるかチェックします．

check_is_dtypelike(dtype)

入力が dtype-like かどうかをチェックします．

check_is_finite(arr, /, *[, name])

配列が有限の値を持つかどうか，つまりNaNやInfの値を持たないかどうかをチェックします．

check_is_greater_than(arr, /, value, *[, ...])

配列の要素がすべてある値より大きいかどうかをチェックします．

check_is_in_range(arr, /, rng, *[, ...])

配列の値がすべて特定の範囲内にあるかどうかをチェックします．

check_is_instance(obj, /, classinfo, *[, ...])

オブジェクトが与えられた型または複数型のインスタンスであるかどうかをチェックします．

check_is_integerlike(arr, /, *[, strict, name])

配列が整数または整数のような浮動小数点値を持つかどうかをチェックします．

check_is_iterable(obj, /, *[, name])

オブジェクトが Iterable のインスタンスであるかどうかをチェックします．

check_is_iterable_of_some_type(iterable_obj, ...)

イテラブルのアイテムがすべて指定された型を持っているかどうかをチェックします．

check_is_iterable_of_strings(iterable_obj, /, *)

イテラブルのアイテムがすべて文字列かどうかをチェックします．

check_is_less_than(arr, /, value, *[, ...])

配列の要素がすべてある値より小さいかどうかをチェックします．

check_is_nonnegative(arr, /, *[, name])

配列の要素がすべて非負かどうかをチェックします．

check_is_number(num, /, *[, name])

オブジェクトが Number のインスタンスであるかどうかをチェックします．

check_is_real(arr, /, *[, name])

配列が実数，すなわち浮動小数点数型か整数型かをチェックします．

check_is_scalar(scalar, /, *[, name])

オブジェクトが実数値のスカラー数かどうかをチェックします．

check_is_sequence(obj, /, *[, name])

オブジェクトが Sequence のインスタンスであるかどうかをチェックします．

check_is_sorted(arr, /, *[, ascending, ...])

配列の値がソートされているかどうかをチェックします．

check_is_string(obj, /, *[, allow_subclass, ...])

オブジェクトが str のインスタンスであるかどうかをチェックします．

check_is_string_in_iterable(string_in, /, ...)

与えられた文字列が反復可能な文字列の中にあるかどうかをチェックします．

check_is_subdtype(arg1, arg2, /, *[, name])

データ型が他の(複数の)データ型のサブタイプであるかどうかをチェックします．

check_is_type(obj, /, classinfo, *[, name])

オブジェクトが与えられた型または複数型のどれかであるかどうかをチェックします．
