pyvista.core.input_validation.validate.validate_array#

validate_array(arr, /, *, must_have_shape=None, must_have_dtype=None, must_have_length=None, must_have_min_length=None, must_have_max_length=None, must_be_nonnegative=False, must_be_finite=False, must_be_real=True, must_be_integer_like=False, must_be_sorted=False, must_be_in_range=None, strict_lower_bound=False, strict_upper_bound=False, reshape_to=None, broadcast_to=None, dtype_out=None, as_any=True, copy=False, to_list=False, to_tuple=False, name='Array')[ソース]#

数値配列が特定の要件を満たしているかチェックし，検証します．

配列が数値であること，特定の形状，データ型を持っていること，ソートされていること，整数のようであること，有限であることなど，特定の要件を満たす値を持っていることを確認するために，配列を検証します．

配列の出力は，形を変えたり，ブロードキャストしたり，ネストされたタプルやリスト配列としてキャストしたり，特定のデータ型にキャストすることもできます．

パラメータ:
arrarray_like

np.ndarray に変換できる任意の形式で，検証される配列．これにはリスト，タプルのリスト，タプル，タプルのタプル，リストのタプル，ndarrayが含まれます．

must_have_shapeint | tuple[int, ...] | list[int, tuple[int, ...]], optional

Check 配列が特定の形状を持っているかチェックします．単一の形状か，任意の形状の list を指定します．整数の場合，配列はその長さで1次元でなければなりません．サイズが可変である次元には -1 を使用する．(デフォルトでは)スカラー値 (0次元) を使用する場合は () を使用します．

must_have_dtypedtype_like | list[dtype_like, ...], optional

Check 配列のデータ型が与えられた dtype を持つかどうかをチェックします． np.dtype オブジェクト，または配列のデータがサブタイプでなければならないdtype-likeな基底クラスを指定します．もし list ならば，配列のデータは指定された dtypes の少なくとも1つのサブタイプでなければなりません．

must_have_lengthint | array_like[int, ...], optional

Check 配列が与えられた長さを持つかどうかを調べます．複数の値が与えられた場合，配列の長さはいずれかの値と一致しなければなりません．

注釈

配列の長さは，（ reshape が None でない場合）配列の形状を変更した後，（ broadcast_to が None でない場合）ブロードキャスト後に決定されます．したがって， length の値は配列の新しい形状を考慮する必要があります．

must_have_min_lengthint, optional

Check 配列の長さがこの値以上の場合．

must_have_max_lengthint, optional

Check 配列の長さがこの値以下の場合．

must_be_nonnegativebool, default: False

Check 配列のすべての要素が負でない場合．

must_be_finitebool, default: False

Check もし配列の全ての要素が有限であれば，つまり infinity でもなく，Not a Number (NaN) でもない場合．

must_be_realbool, default: True

Check もし配列が実数を持つなら，つまりデータ型が整数か浮動小数なら．

must_be_integer_likebool, default: False

Check もし配列の値がinteger-likeであれば（つまり np.all(arr, np.floor(arr)) ）.

must_be_sortedbool | dict, default: False

Check 配列の値がソートされているかチェックします．もし True なら，デフォルトのパラメータでチェックを行います．

  • ascending=True: 配列は昇順にソートされなければなりません．

  • strict=False: 同じ値を持つ連続した要素が許可されます．

  • axis=-1: ソートは配列の最後の軸に沿ってチェックされます．

降順をチェックしたり，厳密な順序を強制したり，別の軸に沿ってチェックしたりするには， check_is_sorted に渡されるキーワード引数を持つ dict を使用します．

must_be_in_rangearray_like[float, float], optional

Check 配列の値が全て特定の範囲内にあるかどうかをチェックします．Rangeは配列のようなもので，2つの要素がそれぞれ許容されるデータの最小値と最大値を指定します．デフォルトでは，範囲の端点は包括的であり，すなわち，値は >= 最小値と <= 最大値でなければなりません． strict_lower_bound および/または strict_upper_bound を使用すると，許容範囲をさらに制限することができます．

..note

Use ``np.inf`` to check for open intervals, e.g.:

* ``[-np.inf, upper_bound]`` to check if values are less
  than (or equal to)  ``upper_bound``
* ``[lower_bound, np.inf]`` to check if values are greater
  than (or equal to) ``lower_bound``
strict_lower_boundbool, default: False

must_be_in_range で指定された範囲の下限を厳密にします．つまり，配列の値は指定された最小値よりも厳密に大きくなければなりません．

strict_upper_boundbool, default: False

must_be_in_range で指定された範囲の上限を厳密にします．つまり，配列の値は指定された最大値よりも厳密に小さくなければなりません．

reshape_toint | tuple[int, ...], optional

np.reshape() を使用して，出力配列を新しい形状にリシェイプします．この形状は元の形状と互換性がなければなりません．もし整数であれば，結果はその長さの1次元配列になります．形状の次元は-1でも構いません．

broadcast_toint | tuple[int, ...], optional

np.broadcast_to() で配列を指定したシェイプの読み取り専用ビューにブロードキャストします．ブロードキャストは整形後に行われます（ reshape_to が None でない場合）．

dtype_outdtype_like, optional

返される配列のデータ型を設定します．デフォルトでは，データ型は入力データから推測されます．以下のようになります．

as_anybool, default: True

np.ndarray のサブクラスがコピーを作成せずに通過できるようにします．

copybool, default: False

もし True ならば，配列のコピーを返します．配列が以下の場合は常にコピーが返されます:

  • ネストされたシーケンス である

  • np.ndarray のサブクラスであり， as_any は False

また， dtype_out 要件を満たすためにコピーを作成することもできます．

to_listbool, default: False

検証済みの配列を list またはネストされた list として返します．スカラ値は常に Number ( int または float ) として返されます． to_tuple=True の場合は無効です．

to_tuplebool, default: False

検証済みの配列を tuple またはネストされた tuple として返します．スカラ値は常に Number ( int または float ) として返されます．

namestr, default: "Array"

エラーメッセージのバリデーションチェックに失敗した場合に使用する変数名．

戻り値:
array_like

検証済みの配列．返されるオブジェクトは次のとおりです:

  • np.ndarray のインスタンス（デフォルト），または

  • 入れ子になった list ( to_list=True の場合)，または

  • 入れ子になった tuple ( to_tuple=True の場合)，または

  • 入力がスカラーの場合は Number (つまり int または float ) ．

例

1次元配列が少なくとも2つの長さを持ち，単調増加（つまり厳密な昇順）であり，ある範囲内にあることを検証します．

>>> import pyvista.core.input_validation as valid
>>> array_in = (1, 2, 3, 5, 8, 13)
>>> rng = (0, 20)
>>> valid.validate_array(
...     array_in,
...     must_have_shape=(-1),
...     must_have_min_length=2,
...     must_be_sorted=dict(strict=True),
...     must_be_in_range=rng,
... )
array([ 1,  2,  3,  5,  8, 13])
